### Phase 4: 動画フリーズ修正（v3.7）

**重大バグ**: 動画を読み込んで開始してもフリーズ、出力動画も固まったまま

**原因分析**:
1. 動画が一時停止したまま再生されていない
2. MediaPipeのタイムスタンプが誤っている（performance.now()使用）
3. processFrameループが動画更新を認識していない

**修正内容**:
1. **動画再生の確実化**
   ```javascript
   // Before: 一時停止チェックのみ
   if (videoElement.paused) {
     await videoElement.play();
   }
   
   // After: 動画ファイルの場合は強制再生
   if (videoFileUrl) {
     videoElement.playbackRate = playbackRate;
     await videoElement.play();
   }
   ```

2. **MediaPipeタイムスタンプ修正**
   ```javascript
   // Before: システム時刻
   const results = await poseLandmarker.detectForVideo(videoElement, currentTime);
   
   // After: 動画の実際の時間
   const results = await poseLandmarker.detectForVideo(videoElement, videoElement.currentTime * 1000);
   ```

3. **一時停止状態チェック追加**
   ```javascript
   if (videoElement.readyState < 2 || (videoFileUrl && videoElement.paused)) {
     animationFrameId = requestAnimationFrame(processFrame);
     return;
   }
   ```

4. **デバッグログ追加**（30フレームごと）

### Phase 5: 多数ユーザー対応（v3.8）

**ユーザーの懸念**:
> 多数ユーザーが使用した際に、サーバーダウンや制限によってアプリが起動しなくなるのではないか

**リスク分析**:
- jsDelivr CDNダウン → 全ユーザー起動失敗
- Google Storageダウン → 姿勢検出モデル取得失敗
- CDNレート制限 → 施設単位でアクセス制限

**解決策: Service Worker実装**

**実装内容**:
1. **Service Worker作成（sw.js）**
   - Cache-First戦略
   - 全外部リソースを自動キャッシュ（13リソース、約10-15MB）
   - オフラインフォールバック

2. **キャッシュ対象**:
   ```
   - MediaPipe vision_bundle.mjs
   - MediaPipe WASM (4ファイル)
   - 姿勢検出モデル (pose_landmarker_lite.task)
   - Chart.js
   - Luxon + Adapter + Streaming Plugin
   - html2canvas
   - jsPDF
   ```

3. **キャッシュ管理機能**:
   - キャッシュ状態インジケーター: "📦 キャッシュ済み (13個)"
   - ネットワーク状態監視（オンライン/オフライン）
   - 手動キャッシュクリア機能
   - 自動バージョン更新通知

**効果**:
| 項目 | Before | After |
|-----|--------|-------|
| 初回起動 | 10-15リクエスト | 10-15リクエスト |
| 2回目以降 | キャッシュ依存 | 0リクエスト（SW） |
| CDNダウン時 | ❌ 起動失敗 | ✅ 正常動作 |
| レート制限時 | ❌ 制限 | ✅ キャッシュから起動 |

**多数ユーザーシナリオ**:
- 100施設×10人＝1000人同時使用
- 初回のみCDNアクセス → 以降完全オフライン
- CDNダウンでも既存ユーザーは100%正常動作

### Phase 6: バグ修正（v3.8.1）

## 🎯 開発で得られた知見

### 1. プライバシー設計
- 匿名ID運用は技術的強制が必須
- ユーザー教育だけでは不十分
- リアルタイムバリデーションが効果的

### 2. オフライン対応
- Service Workerは多数ユーザー環境で必須
- CDNダウンリスクは現実的に発生する
- Cache-First戦略が最適

### 3. 動画処理
- MediaPipeのタイムスタンプは要注意
- performance.now()ではなくvideo.currentTimeを使用
- 一時停止状態の明示的チェックが必要

### 4. UI/UX
- ブラックテーマは眼精疲労を軽減
- リアルタイムフィードバックが重要
- キャッシュ状態の可視化でユーザー安心感向上

### 5. データ管理
- IndexedDB + AES-256-GCMが最適
- エクスポート機能は必須（バックアップ）
- 復号化失敗時のエラーハンドリングが重要
